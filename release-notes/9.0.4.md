# 9.0.4

Solid only. Three things that were quietly not updating now update. Nothing you
call changes, and nothing looks different.

## Callbacks you change after render now take effect

Several Solid components bound a callback prop once, when the component first
rendered, and kept that first function for ever. Hand them a different one later
and the old one still ran.

It affects `NotificationsInbox`'s `onMarkAllRead` and `onViewAll`, and
`DataTable`'s filter-chip remove and column-pin toggle.

This is the sort of thing you would only hit if your handler closes over state
that changes — a `onViewAll` that navigates somewhere depending on the current
tab, say. If you worked around it by keeping the callback identity stable, you
no longer need to.

## DataTable's sort and resize handlers no longer go stale

`DataTable` asked TanStack for its sort and resize handlers once per render and
bound the result. TanStack rebuilds those handlers when column or table state
changes, so a bound one could end up belonging to a table state that no longer
existed.

Sorting and column resizing now look the handler up at the moment you click or
drag. In practice this was hard to trigger; it is simply correct now.

## FormField follows a changed `Field`

`FormField` read its `Field` prop once. If you passed a different `Field`
component afterwards — swapping form instances, for example — it carried on
rendering the original.

## One thing that has NOT changed, and is worth knowing

`Calendar` and `DatePicker` decide which month to show when they first render,
from `selected`. Setting `selected` to a date in a **different month** does not
move the view to that month.

That is deliberate rather than an oversight, and it is now written down as a
decision in the source. If you want the view to follow the selection, say so and
it can change — it is a question about what a date picker should do, not a bug
in how it updates.

## React, vanilla and web-components

Unaffected. None of these bugs existed there: React's JSX re-renders, so a
changed callback prop is picked up for free, and the vanilla binding rebinds on
update.
