# 9.0.3

Housekeeping. Nothing you call changes, and nothing looks different.

## Combobox stops rebuilding its option list on every render

`Combobox` and `MultiCombobox` derived their working list as
`isAsync ? asyncResults : options ?? []`. When you did not pass `options` — the
async case — the `?? []` produced a **new empty array every render**, so every
piece of work that depended on that list redid itself each time, whether or not
anything had actually changed.

You will not see a behaviour difference. It is fewer wasted recomputations in a
component that is often re-rendering on every keystroke.

## Types you import are unchanged, but they are now type aliases

Six prop types were declared as an interface with an empty body:

```ts
export interface SkeletonProps extends React.HTMLAttributes<HTMLDivElement> {}
```

and are now the equivalent alias:

```ts
export type SkeletonProps = React.HTMLAttributes<HTMLDivElement>;
```

Same name, same shape, same import. The one thing an alias cannot do that an
interface can is **declaration merging** — if you were augmenting one of these
from your own code with `declare module`, that will no longer apply. Affects
`SkeletonProps`, `SeparatorProps`, `TextareaProps`, `InputProps`,
`AlertCloseProps` and `BannerCloseProps`. If you were not doing that, there is
nothing here for you.

## Every demo now has a component catalogue, with pictures

The vanilla and web-components demos listed no components at all — their landing
pages were two paragraphs of prose. All four demos now show the same catalogue,
generated from the same navigation the sidebar uses, and each card carries a
thumbnail of the component so the list can be scanned by eye rather than read.

The thumbnails are screenshots of the demos themselves, regenerated on every
deploy, so they cannot quietly drift out of date the way a hand-made set would.

This is the demo site only; nothing ships in the packages.
