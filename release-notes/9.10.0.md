# zen-ui 9.10.0

Nothing breaks. MediaTimeline learns a second trade — overlay-element lanes —
and a click-eating bug in both media components is fixed.

## New: `rangeMode="independent"` on MediaTimeline

9.9.0's MediaTimeline is a trim track: its ranges partition the timeline,
sorted and non-overlapping, and dragging an edge stops at the neighbour.
That is the right physics for cut segments and the wrong physics for the
other lane every editor has — text overlays, stickers, captions: spans that
start and end wherever they like, stack on top of each other, and get
dragged around whole.

```tsx
<MediaTimeline
  duration={30}
  rangeMode="independent"
  ranges={elements}
  activeIndex={selected}            // -1 = nothing selected
  onActiveIndexChange={setSelected} // empty-track click emits -1
  onRangesInput={setElements}
  rangeLabel={(i) => elements[i].text}
  rangeColor={(i) => PALETTE[i % PALETTE.length]}
  minRangeDuration={0.5}
/>
```

What the mode changes:

- **Ranges are free spans.** No neighbour clamps — only the lane's ends and
  the range's own minimum length stop a drag. Overlaps render with array
  order as z-order, later on top.
- **Bars move.** Grab a bar's body and the whole span slides, length
  preserved, the grab point staying under your cursor. The body is focusable
  too — arrow keys nudge it, Shift for full seconds. (Partition mode keeps
  bodies as plain click targets: moving a cut segment through its neighbours
  means nothing, so it does nothing.)
- **Bars say what they are.** `rangeLabel` renders inside the bar — element
  text, a clip name — truncated, and never intercepting the drag.
- **Bars take your colors.** `rangeColor` accepts any CSS color — a hex
  palette entry, an `oklch()`, whatever — and the component derives the fill,
  the selection ring and the handle color from it. `rangeClass` still wins if
  you provide both.
- **Nothing-selected is a state.** Clicking empty track deselects — 
  `onActiveIndexChange(-1)`, the same convention as the DOM's own
  `selectedIndex` — and still seeks.
- The lane is shorter than the filmstrip track by default, the way overlay
  lanes sit in an editor.

The default is `"partition"` and existing timelines behave exactly as before.
Same API in all four bindings; on `<zen-media-timeline>` it is
`range-mode="independent"` with `rangeLabel`/`rangeColor` as JS properties.

## Fixed: the click after a drag ate your seek

In both MediaTimeline and Waveform, the first click on the track after any
drag was silently swallowed — the component armed an internal "ignore the
drag's own click" flag that the drag's click never actually consumed, so
your next real click-to-seek paid for it. One lost click per drag, in every
binding. Fixed everywhere; the drag's own click is still correctly ignored.
