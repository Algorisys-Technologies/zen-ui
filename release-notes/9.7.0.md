# zen-ui 9.7.0

Nothing breaks. This release is about finding the right component — whether the
one looking is you or your AI coding agent.

## New: a Claude Code skill ships in every package

If you build with an AI agent (Claude Code, or anything that reads
`AGENTS.md`), zen-ui now teaches it how to use itself. Every binding package
ships a `skills/zen-ui/` folder — a Claude Code skill that loads automatically
whenever the agent works on UI in a project that depends on zen-ui. Install it
once per project:

```bash
cp -r node_modules/@algorisys/zen-ui-react/skills/zen-ui .claude/skills/
# (or -solid / -vanilla / -web-components — the skill is identical in all four)
```

With it installed, the agent knows things it would otherwise guess at and get
wrong:

- **The full catalogue** — every component with what it is *for*, so it reaches
  for `DateRangePicker` instead of composing two inputs, and for `Stepper`
  instead of hand-rolling a wizard.
- **The setup rules that fail silently** — the mandatory `/styles` import
  (without it everything renders unstyled and nothing errors), the opt-in
  `/preflight`, the optional peer deps behind Chart, RichText, Map and Camera.
- **The binding divergences** — compound parts (`DialogContent`, `TabsList`)
  exist only in React and Solid; vanilla and web-components use one data-driven
  factory per family. Select and Toast differ by design. An agent that learned
  zen-ui from React examples writes broken Solid without this.
- **Theming** — override `--zen-*` custom properties; don't patch generated
  CSS or write `zen-*` utility classes your build can't generate.

The guide is generated from the same source as the demo navigation and checked
in CI, so it cannot drift from the components that actually ship. `AGENTS.md`
(for agents that don't load skills) continues to ship in every package root.

## New: search the component catalogue in every demo

Each demo's sidebar now has a search box. It matches names *and* what a
component is for — typing "wizard" finds Stepper, "busy" finds
PlanningCalendar — and it works the same in all four demos. Groups with no
matches drop out; clearing the box restores the full list. It is built with
zen-ui's own `Search` component, so it is also one more live example.
